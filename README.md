# ecoFAST plots repository

This repository temporarily hosts precomputed plot images used by the ecoFAST Shiny application.

The plots are currently stored on GitHub and accessed remotely by the app.  
In the future, plots may be migrated to a more permanent storage solution.

## Repository structure

Each dataset has its own folder:

```text
<dataset_name>/
├── classif/
├── dynfoot/
├── manifest.csv
└── create_manifest_<dataset_name>.R
```

If classification was run with `str = "aic_asd"` and detection plots were saved, the dataset can additionally contain:

```text
<dataset_name>/detection/
```

Example:

```text
fishbiomass/
├── classif/
├── dynfoot/
├── manifest.csv
└── create_manifest_fishbiomass.R
```

or, when abrupt shift detection plots are available:

```text
fishbiomass/
├── classif/
├── detection/
├── dynfoot/
├── manifest.csv
└── create_manifest_fishbiomass.R
```

## Plot folders

### `classif/`

Contains trajectory classification plots generated by `run_classif()`.

Typical filenames:

```text
ID_aicasd_plot.png
```
ou
```text
ID_aic_plot.png
```

These plots are displayed in the app under:

- Classification → Classification plots

---

### `detection/`

Contains abrupt shift detection plots generated by `run_classif()`.

This folder is only created when:

- `str = "aic_asd"`

Typical filenames:

```text
ID_aicasd_detection.png
```

These plots are displayed in the app under:

- Classification → Abrupt shift detection plots

Datasets generated with `str = "aic"` do not contain detection plots.

---

### `dynfoot/`

Contains dynamical footprint plots generated by `run_dynfoot()`.

Typical filenames:

```text
ID.png
```

These plots are displayed in the app under:

- Dynamical footprint → Dynamical footprint analysis plots

---

## Manifest system

The app does not directly scan repository folders.

Instead, it uses a `manifest.csv` file that lists all available plots and their paths.

The manifest allows the app to:

- retrieve remote plots efficiently;
- map plots to series IDs;
- search plots by ID;
- display plots directly from GitHub or another remote host.

---

## manifest.csv structure

The manifest contains:

| column | description |
|---|---|
| dataset | dataset name |
| series_id | ecological time-series ID |
| plot_type | `classif`, `detection`, or `dynfoot` |
| path | relative path to the image |

Example:

```csv
dataset,series_id,plot_type,path
fishbiomass,COD_1,classif,fishbiomass/classif/COD_1_aicasd_plot.png
fishbiomass,COD_1,detection,fishbiomass/detection/COD_1_aicasd_detection.png
fishbiomass,COD_1,dynfoot,fishbiomass/dynfoot/COD_1.png
```

## Template script

The file:

```text
template_dataset_manifest.R
```

is a generic template used to generate manifest scripts for each dataset.

Before use, replace:

```r
dataset <- #NAME DATASET
```

with the dataset name.

Example:

```r
dataset <- "fishbiomass"
```

The script:

1. scans plot folders;
2. extracts series IDs from filenames;
3. creates a manifest table;
4. writes `manifest.csv`.

---

## Dataset-specific manifest scripts

Each dataset should contain its own manifest generation script.

Example:

```text
create_manifest_fishbiomass.R
```

This script is derived from `template_dataset_manifest.R`.

It generates:

```text
fishbiomass/manifest.csv
```

## Creating a new dataset

To add a new dataset, for example `ram`:

```text
ram/
├── classif/
├── detection/
├── dynfoot/
└── create_manifest_ram.R
```

### Step 1 — add plots

Place plots inside the appropriate folders.

### Step 2 — create manifest script

Copy:

```text
template_dataset_manifest.R
```

to:

```text
create_manifest_ram.R
```

Then modify:

```r
dataset <- "ram"
```

### Step 3 — generate manifest

Run:

```r
source("create_manifest_ram.R")
```

This creates:

```text
ram/manifest.csv
```

## Filename conventions

The manifest system relies on filename conventions to recover series IDs.

### Classification and detection plots

Classification plot filenames depend on the `str` parameter used in `run_classif()`.

Examples:

```text
<ID>_aic_plot.png
<ID>_aicasd_plot.png
```

Detection plots only exist for:

```text
str = "aic_asd"
```

Example:

```text
<ID>_aicasd_detection.png
```

The manifest script extracts the series ID by removing everything after the classification method suffix.

### Dynamical footprint plots

The series ID is extracted from:

```text
<ID>.png
```

## Remote access from the app

The ecoFAST app retrieves plots using:

```r
plot_base_url()
```

and:

```r
remote_plot_url()
```

The app reads:

```text
manifest.csv
```

to map series IDs to image URLs.
